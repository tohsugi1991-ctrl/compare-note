## Should Fix if time permits(改善効果は高いが、現状でも検証は可能)

### SF-1: sourceプリセットに「Claude Code」がない

- **問題**: `CANDIDATE_SOURCE_OPTIONS`(`src/types.ts`)がChatGPT/Claude/Gemini/Copilot/Perplexity/Otherの6択で、「Claude Code」が含まれていない。おすぎさんの実案件(白子町・香取市・IdeaForge含む多くの案件)はClaude Codeで作業していることが多く、実利用頻度が高いsourceが毎回「Other」経由の追加入力を要求される
- **発生した案件**: 3案件すべて(白子町・香取市・IdeaForge)
- **発生回数**: 6候補中3回(50%、案件ごとに必ず1回)
- **ユーザー影響**: 小〜中。ブロッカーではないが、毎回「Otherを選ぶ→もう1つのテキスト欄が出る→入力する」という余分な1ステップが発生し、継続利用時に地味な摩擦として蓄積する
- **修正案**: `CANDIDATE_SOURCE_OPTIONS`に`'Claude Code'`を追加する(`src/types.ts`の配列に1行追加するのみ)
- **想定工数**: 5分未満(型定義の配列に1要素追加のみ、UIロジック・バリデーションは変更不要)
- **MVP仮説への影響**: なし。H1(貼り付け行動の許容度)への影響は軽微(「面倒さ」評価が4→5に上がる程度)であり、仮説検証の可否を左右しない
- **優先度**: 中(Day6でVercelデプロイ作業の合間に着手する程度でよい。Must Fixではないため今回は見送り)

### SF-2: 「回答を追加」フォームが常に比較セクションより上に固定表示される

- **問題**: 候補が2件以上揃って比較・決定だけしたい場面でも、空の「回答を追加」フォーム(source選択+contentテキストエリア)が画面上部に常に表示され、比較カードまでスクロールする必要がある
- **発生した案件**: 3案件すべて(構造上、毎回発生)
- **発生回数**: 3案件×1回(画面を開くたび)
- **ユーザー影響**: 小。スクロール1回分の手間で、致命的ではない
- **修正案**: 候補が1件以上ある場合は「回答を追加」フォームを折りたたみ表示にする、または比較セクションの下に移動する
- **想定工数**: 30分〜1時間程度(UIレイアウトの並び替えとテストの調整が必要なため、SF-1より工数がかかる)
- **MVP仮説への影響**: なし。比較・決定のしやすさ評価(今回5/5)にはまだ影響していない
- **優先度**: 低〜中(Should Fixではあるが、SF-1より優先度は下)

## Do Not Fix yet(便利だが、仮説検証には不要)

### DNF-1: localStorage容量超過(QuotaExceededError)時のエラー表示

- **問題**: `storage.ts`の`saveData`は`localStorage.setItem`の例外を捕捉していない。容量超過時、`ProjectsScreen`の案件作成処理(`handleSubmit`)はtry/catchで囲まれておらず、未処理例外になる可能性がある
- **発生した案件**: 今回のdogfooding(3案件+長文テスト1件)では**発生しなかった**
- **発生回数**: 0回(実際の使用量ではlocalStorageの容量[通常5〜10MB]に到達する見込みが薄い)
- **ユーザー影響**: 現時点では低い(想定利用規模=2〜6案件、候補数件程度では到達しない)
- **修正案**: `saveData`をtry/catchで囲み、呼び出し元に分かりやすいエラーメッセージを返す
- **想定工数**: 1〜2時間(呼び出し元全箇所のエラーハンドリング見直しが必要)
- **MVP仮説への影響**: なし。H1〜H3のいずれの検証にも今回影響しなかった
- **優先度**: 低(初期10人への公開後、実際に問題が報告されてから対応する)

### DNF-2: 極端に長いテキスト貼り付け時のエラー表示

- **問題**: 7day_plan Day5節に記載されていた想定タスク。実際に約16,000文字を貼り付けて検証した結果、既存の`max-h-80 overflow-y-auto`(候補カードの内部スクロール)により**レイアウト崩れもエラーも発生しなかった**

- **発生回数**: 検証1回、問題は発生せず
- **ユーザー影響**: なし(現状の実装で十分に安全)
- **修正案**: 対応不要と判断。追加のエラー表示・文字数制限は設けない
- **想定工数**: 0(実施しない)
- **MVP仮説への影響**: なし
- **優先度**: 対応不要としてクローズ
