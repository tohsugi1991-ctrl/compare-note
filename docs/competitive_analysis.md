# Day7 Light Validation — Phase2: 競合比較

同じ作業(「複数AIの回答を横に並べて比較し、採用理由を1〜3行で記録し、後日その決定を思い出す」)を、Notion・Obsidian・Apple Notes・VS Code Markdown・Google Docsで行った場合とCompare Noteを比較する。比較対象の作業は[day7_light_validation.md](./day7_light_validation.md)の3案件と同じ粒度(候補2件・200〜400文字/件・採用理由100〜200文字)を想定する。

## 1. ツールごとの実行速度・価値・弱点

| ツール | 入力速度 | 比較のしやすさ | 決定記録 | 再訪速度 | 速い/遅い | 独自の価値 | 負けている点 |
|---|---|---|---|---|---|---|---|
| **Compare Note** | やや遅い(専用フォーム、source選択という追加ステップ) | **速い**(2件は自動で2カラムグリウ、選ぶだけで比較できる) | **速い**(reason必須のフォームが常に出るため書き忘れない) | **最速**(決定済みの案件は一覧で一目、開けば決定カードが最上部) | 総合では最速 | 「採用理由を書かないと決定を保存できない」という強制力。案件横断で「決定済み/未決定」が一覧性を持つ | 検索・タグ・複数端末同期・共有が一切ない。装飾(太字・画像・表)も使えない |
| **Notion** | 普通(データベース+テンプレートを事前に作れば速いが、その場で作ると遅い) | 普通(トグルやカラムで再現できるが、毎回自分でレイアウトを組む必要がある) | 遅い(「採用理由」を書く欄を自分で用意しないと、書かずに済ませてしまいがち) | 普通(検索・タグは強いが、「この案件は決定済みか」は自分でステータスプロパティを運用しないと分からない) | 初回セットアップの分だけ遅い、運用を仕組み化すれば追いつく | データベースとしての横断検索・フィルタ・他ページとの連携 | 「比較→理由記録」という型そのものは提供されない。型を自分で維持し続ける必要がある |
| **Obsidian** | 普通(ローカルMarkdown、貼り付け自体は速い) | 遅い(左右分割ペインを手動操作しないと横並びにならない) | 遅い(理由を書く欄が強制されないため後回しにしがち) | 速い(バックリンク・グラフビューでの再発見は強いが、「なぜ選んだか」の即答性はない) | 比較・決定の型が無い分、体感は遅い | ローカルファイル・バックリンクによる長期的なナレッジ蓄積 | 「今この案件は決定済みか」を一覧で判断する仕組みがない |
| **Apple Notes** | **最速**(ただ貼り付けるだけ) | 遅い(構造が一切ないため、候補が増えるほど縦に長い1枚のメモになる) | 遅い(理由を書く習慣がなければ書かずに終わる) | 遅い(決定済みかどうかはメモ全文を読み返さないと分からない) | 入力だけなら最速、比較・決定・再開は最も遅い | 手軽さ・iPhoneとの連携 | 構造・検索性・「なぜ選ばなかったか」を残す仕組みが皆無 |
| **VS Code Markdown** | 速い(使い慣れたエディタでの入力自体は速い。おすぎさんの実務環境そのもの) | 遅い(Markdownテーブルや手動整形をしないと横並びにならない) | 普通(git履歴に「いつ書き換えたか」は残るが、「なぜ選んだか」という理由欄は自分で用意する必要がある) | 普通(ファイル名・ディレクトリを自分で運用すれば速いが、案件横断の一覧性はエディタの外の話になる) | 入力は速いが比較・決定の型がない分、総合では中位 | 既存の実務フロー(Claude Code)とシームレスに繋がる、Git管理下での差分追跡 | 「決定済み/未決定」を横断して見る画面がない。理由を書く強制力もない |
| **Google Docs** | 普通(ブラウザでの貼り付け自体は速いが、複数案件をまたぐ管理はしづらい) | 遅い(表を手動で作れば横並びにできるが手間) | 普通(コメント機能で理由を残すことはできるが、専用の型ではない) | 遅い(検索はできるが、「決定済みか」は文書を開いて読まないと分からない) | チームで共有する前提でなければ総合的に遅い | チームでの共同編集・コメント・共有のしやすさ | ローカル完結ではない(クラウド前提)、個人の一次判断の速さでは他ツールに劣る |

## 2. 観点別の勝敗

- **入力の生の速さ**では Apple Notes が最も速い(ただの貼り付け)。Compare Noteはsource選択という1ステップ分、単純な貼り付けより遅い。
- **比較のしやすさ**ではCompare Noteが最も速い。他ツールはすべて「横並びにするために自分でレイアウトを作る」という一手間が必要で、この一手間は案件ごとに繰り返し発生する。
- **決定を書き残す確実性**ではCompare Noteが唯一「理由なしでは保存できない」という強制力を持つ。他ツールは書くこと自体は可能だが、書かなくても成立してしまうため、忙しい時ほど省略されるリスクがある。
- **再訪時の速さ**ではCompare Noteが最も速い(決定済みバッジ+決定カードの自動表示)。Obsidianのバックリンクは長期的な発見には強いが、「今すぐこの案件の結論だけ知りたい」という場面ではCompare Noteの方が速い。
- **チーム共有・複数端末**ではNotion/Google Docsが明確に勝る。Compare Noteはこの軸を意図的に持たない(1人・1ブラウザ完結の設計)。
- **検索・蓄積量が増えた場合の運用**ではNotion/Obsidianが勝る。Compare Noteは案件数が数十件を超えた時点で一覧性が崩れる設計(検索・タグ・フォルダを持たない)。

## 3. なぜCompare Noteを使うべきか(1文)

**他のツールは「メモを取れる」だけだが、Compare Noteは画面の構造そのものが「複数候補を横に並べさせ、理由を書かないと先に進ませず、次に開いた瞬間に結論を思い出させる」ように設計されているため、同じ比較・決定作業を他ツールで行うより速く終わり、かつ「なぜ他の案を選ばなかったか」まで含めて後から確実に思い出せる、唯一のツールである。**
