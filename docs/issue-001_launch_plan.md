# ISSUE-001 公開計画:公開方法・初期ユーザー獲得・収益化の検討

---

## 公開方法

無料または低コストで公開できる静的ホスティングを使う。React+Vite+TypeScriptはビルドすると静的ファイル一式になり、サーバー処理を持たないため以下のいずれでも公開できる。

| 選択肢 | 特徴 | このMVPでの位置づけ |
|---|---|---|
| **Vercel** | Viteプロジェクトをほぼ無設定でデプロイ可能。無料枠で十分、独自ドメインも無料枠内で設定可能 | **採用**。7日という時間制約の中でセットアップの手間が最も少ない |
| Netlify | Vercelとほぼ同等。ドラッグ&ドロップでのデプロイも可能 | 代替候補。Vercelで問題が出た場合のフォールバック |
| GitHub Pages | 追加の外部アカウント登録が不要(GitHubのみで完結) | 代替候補。ただしSPAのルーティング設定がやや面倒なため優先度は下げる(本MVPはルーティングライブラリを使わない設計のため実際には支障は少ない) |

### localStorageを使う場合の制約(再掲・公開時にユーザーへ明示する)

- データはブラウザ単位。PCとスマホ、別のブラウザでは共有されない
- キャッシュ削除・シークレットモードでは消える
- エクスポート・バックアップ機能はv1に含まない

これらは告知文・アプリ内表示の両方で正直に伝える(詳細は[issue-001_data_schema.md](./issue-001_data_schema.md))。

## 初期ユーザー獲得:最初の10人

広告は使わない。以下の4チャネルから、Day7に合計10人へ届ける。

| チャネル | 人数目安 | 訴求の出し方 |
|---|---|---|
| 知人(AIを複数使う人) | 3〜4人 | LINE/Slackで個別DM。コンサル業務・地方自治体向け提案作成・旅行や育児ブログの執筆など、複数AIを使いながら案件を進めている知人に直接送る |
| X(個人アカウント) | 2〜3人からの反応を想定 | 「7日間で作った」開発ログ形式の投稿。スクリーンショット付きで、比較→決定→再訪確認の一連を見せる |
| note | 1〜2人からの反応を想定 | JTBD分析からMVPに至る過程を記事化し、末尾にアプリへのリンクを置く。地方自治体向け提案業務など、既に発信している領域と地続きの読者に届きやすい |
| 個人開発者・Claude Code利用者コミュニティ | 2〜3人 | 参加しているDiscord/Slackコミュニティ、またはX上のClaude Code関連スレッドへの投稿。「個人開発者自身が複数AIを使う場面」で刺さる訴求にする |

### 訴求文(下書き)

> ChatGPT・Claude・Geminiを行ったり来たりしながら仕事してませんか。
> 「どの回答を採用したか、なぜか」を1〜3行で記録しておくだけの、ログイン不要のローカルWebツールを作りました。
> データは自分のブラウザにしか保存されないシンプルな作りです。案件を再開したときの「あれ、どこまでやったっけ」を減らせるか試しています。よければ触ってみて、感想を聞かせてもらえると嬉しいです。→ [URL]

10人への案内は個別に送る(一斉投稿だけに頼らない)。理由: 初期フィードバックの質を担保するため、少なくとも知人3〜4人には直接使ってもらい、率直な感想(特にKill Criteriaに関わる「面倒」「Notionで十分」という声)を拾いやすくする。

## 収益化(v1では課金しない。将来の検証のための整理のみ)

### 想定価格

- 買い切り ¥980、または将来クラウド同期等の継続コストが発生する機能を追加する場合は月額¥400程度のプレミアム層を検討
- どちらが適切かはこの7日間のMVPでは検証しない。ローカル保存のみの現行仕様はサーバー運用コストがほぼゼロのため、無理に月額課金にする必然性が薄く、買い切りの方が自然だが断定はしない

### 無料版と有料版の境界(案)

- v1(今回)は完全無料・無制限
- 将来の境界候補: 「案件数の上限」(例: 無料は5案件まで)、または「クラウド同期・複数デバイス間の共有」を有料化ラインにする案。ローカル保存のみなら無料のままで十分という前提と矛盾しない設計にする

### ユーザーが支払う理由(仮説)

JTBD_report 14節が指摘する通り、使うほど「案件ごとの比較・判断根拠データ」が蓄積し、再利用ステージの満足度が上がる。決定記録が資産になるほど、失いたくない・上限を超えて使いたいという動機が生まれる可能性がある。これ自体もMVPで検証すべき仮説の1つであり、7日間では検証しない。

### 課金前に確認すべき数値

1. 無料版での案件作成数の分布(5件を超えて使い続けるヘビーユーザーがどれだけいるか)
2. 決定記録の完了率が安定して60%以上を維持できているか([issue-001_validation_plan.md](./issue-001_validation_plan.md)の指標)
3. 「今後も使いたい」と答えたユーザーが、実際に7日を超えて使い続けているか(継続日数)

これらが確認できるまで課金導線は作らない。収益予測はここでは断定しない。
